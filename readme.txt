# CS 210 Portfolio Project – Corner Grocer Item Tracking Program

## Project Summary

For my portfolio project, I selected the Corner Grocer Item Tracking Program. The purpose of this project was to create a C++ application that reads grocery purchase data from a file, counts how many times each item was purchased, and allows the user to search for an item's frequency, display all purchase frequencies, and generate a histogram. This project demonstrates file input/output, data structures, object-oriented programming, and user interaction.

## What I Did Well

One of the strongest parts of my project was organizing the code into separate classes and functions, making it easier to understand and maintain. I also included comments throughout the program and created a menu-driven interface that is simple for users to navigate. The program successfully reads data from a file and produces accurate results.

## Where I Could Improve

If I were to improve this project, I would add more input validation to prevent invalid menu selections and unexpected user input. I would also improve error handling when opening files and make the program more flexible by allowing users to choose different input files instead of relying on a single filename. These improvements would make the application more secure, reliable, and user-friendly.

## Challenges and How I Overcame Them

The most challenging part of this project was working with file input and using a map to store and count item frequencies. It took some practice to understand how to read each item from the file and update its count correctly. I overcame these challenges by reviewing course materials, testing my code with different input files, and debugging step by step until the output matched the expected results.

## Transferable Skills

This project strengthened several skills that I will use in future coursework and software development projects. These include object-oriented programming, working with C++ classes, using maps and other data structures, file input/output, writing modular code, and debugging programs. These skills will be valuable in future programming courses and professional software development.

## Maintainability, Readability, and Adaptability

I made this program maintainable by separating responsibilities into different functions and using meaningful variable and function names. I included comments where appropriate and followed consistent formatting throughout the code. This design makes it easier to update, expand, or modify the program in the future without affecting the overall functionality.
